Chapter 01 · Foundation

The Terminal

Why this matters The terminal is where you'll spend half your time. Windows ships with an old one — we want the modern one and the modern shell that runs inside it. Five minutes here saves you a lifetime of papercuts.

Step 01 / AInstall Windows Terminal

Windows Terminal is the modern command-line app from Microsoft. Tabs, themes, copy-paste that actually works. It's free.

Microsoft Store ↗

Step 01 / BInstall PowerShell 7 (latest)

The shell that came with Windows is Windows PowerShell 5.1. We want PowerShell 7+ — faster, cross-platform, modern syntax. Open the Terminal you just installed and paste:

winget install --id Microsoft.PowerShell --source winget

Close and reopen the Terminal afterwards.

Step 01 / CMake PowerShell 7 the default

In Windows Terminal, click the ▼ arrow next to the new-tab button → Settings → under Default profile select PowerShell (not "Windows PowerShell" — they look identical, but the one without "Windows" is v7). Save.

Plain English"Terminal" is the window. "Shell" is the language you type into it. Windows Terminal + PowerShell 7 = the modern pairing. From here on, every command in this book is pasted into this window.

Step 01 / DInstall winget (probably already there)

winget is Windows' built-in package manager — think "App Store for the command line." It comes with Windows 11. Verify it works:

winget --version

If you see a version number, you're set. If not, install App Installer ↗ from the Microsoft Store.

Chapter 02 · Workspace

The Editor

Why this matters A code editor is your canvas. We use one with built-in AI — Cursor — because it lets you describe what you want and watches you write. This is the heart of "vibe coding."

Step 02 / AInstall Cursor

Cursor is a fork of VS Code with native AI. Install it directly:

cursor.com/download ↗

Or via terminal:

winget install --id Anysphere.Cursor

Step 02 / B(Optional) VS Code as a backup

If you ever want the plain non-AI editor too:

winget install --id Microsoft.VisualStudioCode

Step 02 / CFirst-launch settings

Open Cursor. When it asks:

  • Sign in with GitHub (we'll create that account next chapter — skip for now, come back)
  • Color theme: pick anything that makes you happy
  • Install the "code" command in PATH when prompted — this lets you type cursor . in any folder to open it
TipPress Ctrl + L in Cursor to open the AI sidebar. Press Ctrl + K while editing to ask the AI to write or rewrite the highlighted code.

Chapter 03 · Version Control

Git & GitHub

Why this matters Git is a time machine for your code — every save is a checkpoint you can return to. GitHub is where those checkpoints live in the cloud, shared with collaborators, deployed to servers, and tied to your portfolio.

Step 03 / AInstall Git

winget install --id Git.Git -e

Close and reopen Terminal afterwards. Verify:

git --version

Step 03 / BTell Git who you are

Replace the name and email with your own (use the same email you'll use for GitHub):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

Step 03 / CCreate a GitHub account

github.com/signup ↗

Pick a username you'll be okay with for years — it shows up in every URL. Verify your email before continuing.

Step 03 / DInstall GitHub CLI & sign in

The GitHub CLI (gh) handles all the authentication headaches for you.

winget install --id GitHub.cli

Close and reopen Terminal. Then:

gh auth login

Choose: GitHub.com → HTTPS → Yes (auth Git with credentials) → Login with a web browser. Copy the one-time code, paste in the browser, done.

What just happenedYou can now git push code to GitHub from any folder without ever typing a password. The CLI also lets you create repos with gh repo create and open PRs from the terminal.

Chapter 04 · Engine

The Runtime

Why this matters Your code needs something to run it. For modern web apps that "something" is Node.js (and increasingly its faster cousin, Bun). Package managers like pnpm and bun handle the libraries your projects pull in.

Step 04 / AInstall Node.js (LTS)

LTS = "Long Term Support," the boring stable version everyone uses in production.

winget install --id OpenJS.NodeJS.LTS

Close and reopen Terminal. Verify:

node --version
npm --version

Step 04 / BInstall Bun

Bun is a faster, modern alternative to Node + npm. We'll use it for many projects. One-liner:

powershell -c "irm bun.sh/install.ps1 | iex"

Close and reopen Terminal. Verify:

bun --version

Step 04 / CInstall pnpm

pnpm is another package manager — disk-space-efficient, used by lots of repos. Easy way:

npm install -g pnpm
pnpm --version
Plain EnglishThink of npm, pnpm, and bun as three different grocery delivery services for code libraries. You'll use whichever the project you cloned uses — they're mostly interchangeable, but you can't mix them in one project.

Chapter 05 · The Co-pilot

Claude Code

Why this matters Claude Code is Anthropic's AI agent that lives in your terminal. It reads your files, runs your tests, edits code, and pushes commits — all from natural-language instructions. The single biggest leverage tool in this entire stack.

Step 05 / AGet Claude Pro

Pro is $20/month and unlocks the high-quality models with generous usage limits. Subscribe at:

claude.ai/upgrade ↗

While you're there, sign in to claude.ai ↗ with the same account — that's the chat interface for non-coding work.

Step 05 / BInstall Claude Code

npm install -g @anthropic-ai/claude-code

Verify:

claude --version

Step 05 / CFirst run & sign-in

Move into any folder (we'll create one in Chapter 8). Then:

claude

It'll open a browser window — sign in with your Claude Pro account. From now on, just typing claude in any folder gives you an AI that knows your project.

Step 05 / DThe three magic prompts

  • "What is this codebase?" — get oriented when you join an unfamiliar repo
  • "Build me X" — describe a feature, watch it implement, review the diff
  • "Why is this broken?" — paste the error, let it debug
EtiquetteAlways read the diff before approving a change. Claude is fast and confident — that combination occasionally produces fast, confident mistakes. Trust, but verify.

Chapter 06 · Hosting

Vercel

Why this matters Vercel takes a folder of code and turns it into a live website with a real URL — in seconds. Push to GitHub, Vercel deploys automatically. The free tier covers hobby projects forever.

Step 06 / ASign up for Vercel

Use the "Continue with GitHub" option — it links the two services so deploys happen automatically when you push code.

vercel.com/signup ↗

Step 06 / BInstall the Vercel CLI

npm install -g vercel
vercel login

Choose Continue with GitHub. Browser opens, click confirm, done.

Step 06 / CDeploy any folder, instantly

Inside a project folder:

vercel

Answer the prompts (defaults are fine for the first run). When it's done, it gives you a public URL. That's it. You're on the internet.

The dreamOnce a repo is "linked" to Vercel, every git push auto-deploys. You write code → save → push → site updates. There is no further ceremony.

Chapter 07 · Backend

Supabase

Why this matters Supabase is the backend you'd otherwise have to build from scratch — a Postgres database, user authentication, file storage, and realtime subscriptions, all bundled with a friendly dashboard. Free tier is plenty for starting out.

Step 07 / ASign up

Use "Continue with GitHub" again so everything stays linked.

supabase.com/sign-up ↗

Step 07 / BCreate your first project

From the dashboard:

  • Click New Project
  • Pick a name and a strong database password (save it somewhere safe — you'll need it)
  • Choose the region closest to you
  • Wait ~2 minutes while it provisions

Step 07 / CInstall the Supabase CLI

Easiest way on Windows is via Scoop, but npm works too:

npm install -g supabase
supabase login

Step 07 / DGrab your API keys

In the Supabase dashboard for your project: Project Settings → API. You'll need:

  • Project URL — public, identifies your database
  • anon (public) key — safe to use in frontend code
  • service_role key — never commit this to git, server-side only
Heads-upAnything that ends in _role or is labeled secret goes in a file called .env.local — never in code, never in GitHub. Supabase will yell at you if you slip up. So will Claude.

Chapter 08 · Liftoff

Your First Vibe

Why this matters A 10-minute walkthrough of the entire loop: idea → code → repo → live site. Once you've done it once, every project after this is muscle memory.

Step 08 / AMake a projects folder

mkdir $HOME\dev cd $HOME\dev

Step 08 / BCreate a Next.js app

Next.js is the most-used React framework — Vercel made it, so it deploys there in one click.

npx create-next-app@latest my-first-vibe

Accept all the defaults (TypeScript, Tailwind, App Router — yes to everything).

cd my-first-vibe

Step 08 / COpen it in Cursor

cursor .

Step 08 / DStart the dev server

Inside the Cursor terminal (Ctrl + ` to open it):

npm run dev

Open localhost:3000 ↗. Your app is live on your machine.

Step 08 / EVibe with Claude

Open a second terminal tab and run:

claude

Try: "Replace the home page with a cosmic-themed landing page for an app called 'StarMail.' Use the existing Tailwind setup." Watch it work. Approve the diff. Refresh the browser.

Step 08 / FPush to GitHub

git add . git commit -m "first vibe" gh repo create --source=. --public --push

Step 08 / GDeploy to Vercel

vercel

Hit enter through the prompts. Sixty seconds later, you have a public URL. Send it to a friend.

Git Essentials

git statusWhat's changed since the last save?

git pushSend snapshots to GitHub
git pullGet the latest from GitHub

GitHub CLI

gh repo createCreate a new repo from current folder
gh repo clone user/nameDownload someone else's project
gh pr createOpen a pull request

Package Managers

pnpm iSame, but with pnpm
bun installSame, but with bun (fastest)
