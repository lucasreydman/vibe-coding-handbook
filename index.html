Chapter 01 · Foundation

The Terminal

Why this matters The terminal is where you'll spend half your time. Windows ships with an old one — we want the modern one and the modern shell that runs inside it. Five minutes here saves you a lifetime of papercuts.

Step 01 / AInstall Windows Terminal

Windows Terminal is the modern command-line app from Microsoft. Tabs, themes, copy-paste that actually works. It's free.

Microsoft Store ↗

Step 01 / BInstall PowerShell 7 (latest)

The shell that came with Windows is Windows PowerShell 5.1. We want PowerShell 7+ — faster, cross-platform, modern syntax. Open the Terminal you just installed and paste:

winget install --id Microsoft.PowerShell --source winget

Close and reopen the Terminal afterwards.

Step 01 / CMake PowerShell 7 the default

In Windows Terminal, click the ▼ arrow next to the new-tab button → Settings → under Default profile select PowerShell (not "Windows PowerShell" — they look identical, but the one without "Windows" is v7). Save.

Plain English"Terminal" is the window. "Shell" is the language you type into it. Windows Terminal + PowerShell 7 = the modern pairing. From here on, every command in this book is pasted into this window.

Step 01 / DInstall winget (probably already there)

winget is Windows' built-in package manager — think "App Store for the command line." It comes with Windows 11. Verify it works:

winget --version

If you see a version number, you're set. If not, install App Installer ↗ from the Microsoft Store.

Chapter 02 · Version Control & Identity

Git & GitHub

Why this matters Git is a time machine for your code — every save is a checkpoint you can return to. GitHub is where those checkpoints live in the cloud. We set this up first because the rest of the stack — VS Code, Vercel, Supabase, Copilot — all let you "Sign in with GitHub." Make this account once and reuse it everywhere.

Step 02 / AInstall Git

winget install --id Git.Git -e

Close and reopen Terminal afterwards. Verify:

git --version

Step 02 / BTell Git who you are

Replace the name and email with your own (use the same email you'll use for GitHub):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main

Step 02 / CCreate a GitHub account

github.com/signup ↗

Pick a username you'll be okay with for years — it shows up in every URL. Verify your email before continuing.

Step 02 / DInstall GitHub CLI & sign in

The GitHub CLI (gh) handles all the authentication headaches for you.

winget install --id GitHub.cli

Close and reopen Terminal. Then:

gh auth login

Choose: GitHub.com → HTTPS → Yes (auth Git with credentials) → Login with a web browser. Copy the one-time code, paste in the browser, done.

What just happenedYou can now git push code to GitHub without ever typing a password. The CLI also creates repos (gh repo create) and opens PRs from the terminal. Hold onto this account — VS Code, Vercel, and Supabase all let you sign in with it in the next few chapters.

Chapter 03 · Workspace

The Editor

Why this matters A code editor is your canvas. We use VS Code because it's free, used by half the industry, and a few well-chosen extensions give it the same AI superpowers people pay $20/mo for in Cursor.

Step 03 / AInstall VS Code

code.visualstudio.com ↗

Or via terminal:

winget install --id Microsoft.VisualStudioCode

Step 03 / BFirst-launch settings

  • Sign in with GitHub when prompted — use the account you set up in Chapter 2. This syncs your settings across devices and is what unlocks Copilot's free tier in the next step.
  • Pick a color theme that doesn't fatigue your eyes
  • When asked, install the "code" command in PATH — that's what lets you type code . in any folder to open it

Step 03 / CThe AI extensions

Hit Ctrl + Shift + X to open the Extensions panel. Install these three first:

  • Claude Code (Anthropic, official) — runs the Claude Code agent inside VS Code's sidebar with a hotkey. The headline tool.
  • GitHub Copilot — autocompletes code as you type. Free tier gives you ~2,000 completions and 50 chats per month — plenty for a hobby project. Activates automatically because you signed in with GitHub above.
  • GitHub Copilot Chat — the in-editor chat panel powered by Copilot.

That trio gets you 95% of what Cursor does, free.

Step 03 / DA small kit, on purpose

Most VS Code extension lists are written for people typing every line themselves. You're not — Claude is. So skip the autocomplete + snippet maximalism and install these few, which actually pull weight when you're reading code instead of writing it:

  • Error Lens — shows errors in red, inline next to the offending line. The single most useful extension when you're not a typing coder. You'll instantly see when something's broken.
  • Pretty TypeScript Errors — translates TypeScript's cryptic walls of error text into readable English. Makes "what's wrong, and what do I tell Claude" obvious.
  • Prettier — Code formatter — auto-formats every save so AI-generated code stays consistent and the diffs you read are clean.
  • ESLint — flags real bugs and broken patterns. Pair with Error Lens above to actually see them.
  • Material Icon Theme — file-type icons in the explorer. Helps you eyeball a project's structure at a glance.

That's it. If you ever start hand-coding and want typing helpers (Tailwind IntelliSense, snippet packs, Auto Rename Tag), grab them then — they're noise until then.

Step 03 / ETwo settings to flip

Press Ctrl + , (comma) to open Settings. Search and enable:

  • "Format On Save" — turn on. Prettier runs every time you hit save.
  • "Default Formatter" — set to Prettier - Code formatter.
Cursor as an alternativeIf you decide later that you want AI even more deeply baked in, Cursor ↗ is a paid VS Code fork with native AI editing. Your VS Code extensions all still work there. But you don't need it to start — VS Code + Claude Code + Copilot covers the same ground for free.

Chapter 04 · Engine

The Runtime

Why this matters Your code needs something to run it. For modern web apps that "something" is Node.js (and increasingly its faster cousin, Bun). Package managers like pnpm and bun handle the libraries your projects pull in.

Step 04 / AInstall Node.js (LTS)

LTS = "Long Term Support," the boring stable version everyone uses in production.

winget install --id OpenJS.NodeJS.LTS

Close and reopen Terminal. Verify:

node --version
npm --version

Step 04 / BInstall Bun

Bun is a faster, modern alternative to Node + npm. We'll use it for many projects. One-liner:

powershell -c "irm bun.sh/install.ps1 | iex"

Close and reopen Terminal. Verify:

bun --version

Step 04 / CInstall pnpm

pnpm is another package manager — disk-space-efficient, used by lots of repos. Easy way:

npm install -g pnpm
pnpm --version
Plain EnglishThink of npm, pnpm, and bun as three different grocery delivery services for code libraries. You'll use whichever the project you cloned uses — they're mostly interchangeable, but you can't mix them in one project. This book defaults to npm because it ships with Node and works everywhere; reach for bun or pnpm when you want speed or smaller installs.

Chapter 05 · The Co-pilot

Claude Code

Why this matters Claude Code is Anthropic's AI agent that lives in your terminal. It reads your files, runs your tests, edits code, and pushes commits — all from natural-language instructions. The single biggest leverage tool in this entire stack.

Step 05 / AGet Claude Pro

Pro is $20/month and unlocks the high-quality models with generous usage limits. Subscribe at:

claude.ai/upgrade ↗

While you're there, sign in to claude.ai ↗ with the same account — that's the chat interface for non-coding work.

Step 05 / BInstall Claude Code

npm install -g @anthropic-ai/claude-code

Verify:

claude --version

Step 05 / CFirst run & sign-in

Move into any folder (we'll create one in Chapter 9). Then:

claude

It'll open a browser window — sign in with your Claude Pro account. From now on, just typing claude in any folder gives you an AI that knows your project.

Step 05 / DThe three magic prompts

  • "What is this codebase?" — get oriented when you join an unfamiliar repo
  • "Build me X" — describe a feature, watch it implement, review the diff
  • "Why is this broken?" — paste the error, let it debug

Step 05 / EInside VS Code

If you installed the Claude Code extension back in Chapter 3, you can also launch the agent without leaving the editor. Open the command palette with Ctrl + Shift + P and type "Claude Code: Start" — same agent, sidebar instead of a terminal. Use whichever feels better moment to moment.

Step 05 / FMake it your own — CLAUDE.md

Claude Code reads a file called CLAUDE.md at the start of every conversation. This is where you bend Claude to your habits — enforce planning before coding, demand verification before "done," tell it to find root causes instead of patching symptoms, route hard tasks to Opus, etc. Two places it looks:

  • $HOME\.claude\CLAUDE.md — global; applies to every project on your machine
  • ./CLAUDE.md in a project root — per-project; layered on top of the global one for repo-specific rules

I made you a starter file — a generalized version of mine. Drop it into the global slot:

mkdir $HOME\.claude -Force iwr https://lucasreydman.github.io/vibe-coding-handbook/CLAUDE.md -OutFile $HOME\.claude\CLAUDE.md

Browse it first if you want: CLAUDE.md ↗

The headline rules: plan before coding, verify before "done", find root causes — don't paper over symptoms, keep diffs small, match the model to the task. Edit it whenever you find yourself correcting Claude on the same thing twice — that correction belongs in the file so you don't have to repeat it.

EtiquetteAlways read the diff before approving a change. Claude is fast and confident — that combination occasionally produces fast, confident mistakes. Trust, but verify.

Vercel

Why this matters Vercel takes a folder of code and turns it into a live website with a real URL — in seconds. Push to GitHub, Vercel deploys automatically. The free tier covers hobby projects forever.

Chapter 07 · Backend

Supabase

Why this matters Supabase is the backend you'd otherwise have to build from scratch — a Postgres database, user authentication, file storage, and realtime subscriptions, all bundled with a friendly dashboard. Free tier is plenty for starting out.

Step 07 / ASign up

Use "Continue with GitHub" again so everything stays linked.

supabase.com/sign-up ↗

Step 07 / BCreate your first project

From the dashboard:

  • Click New Project
  • Pick a name and a strong database password (save it somewhere safe — you'll need it)
  • Choose the region closest to you
  • Wait ~2 minutes while it provisions

Step 07 / CInstall the Supabase CLI

Install via npm:

npm install -g supabase
supabase login

Step 07 / DGrab your API keys

In the Supabase dashboard for your project: Project Settings → API. You'll need:

  • Project URL — public, identifies your database
  • anon (public) key — safe to use in frontend code
  • service_role key — never commit this to git, server-side only
Heads-upAnything that ends in _role or is labeled secret goes in a file called .env.local — never in code, never in GitHub. Supabase will yell at you if you slip up. So will Claude.

Chapter 08 · Configuration

Secrets & Config

Why this matters Apps need keys — Supabase, Stripe, Resend, AI services. Some are safe to ship to the browser. Others, if leaked, can get you billed for someone else's crypto-mining habit. This chapter is how to keep them straight.

Step 08 / AThe .env.local file

In any Next.js project, create a file called .env.local in the project root (next to package.json). Format:

NEXT_PUBLIC_SUPABASE_URL=https://abcd.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGciOi... SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOi... STRIPE_SECRET_KEY=sk_test_... RESEND_API_KEY=re_...

One key per line. No quotes, no spaces around the =. In code, read them with process.env.STRIPE_SECRET_KEY.

Step 08 / BPublic vs server-only

  • Public-safe — Supabase anon key, Stripe publishable key. Designed to be shipped to the browser. Prefix with NEXT_PUBLIC_ to expose them on the frontend.
  • Server-only — Supabase service_role, Stripe secret key, Resend API key, OpenAI/Anthropic keys. These bypass security. Never log them, never put them in client-side code, never commit them.

Step 08 / C.gitignore — the no-fly list

This is a file that tells Git to pretend certain files don't exist. Next.js auto-generates one with .env* and node_modules already in it — do not delete those lines. If you ever start a project from scratch, your .gitignore needs at minimum:

node_modules .env .env.local .next .DS_Store

Step 08 / DSync env vars to Vercel

Your local .env.local isn't deployed — it stays on your machine. To give your live site its keys:

vercel env add STRIPE_SECRET_KEY

It'll prompt for the value and which environments (development / preview / production). Or use the Vercel dashboard → Project → Settings → Environment Variables. To pull cloud values down to your machine:

vercel env pull .env.local
If you commit a secret by accidentRotate it immediately at the source — Supabase dashboard → revoke key, Stripe dashboard → roll secret, Resend → delete API key. Removing it from Git history is fiddly; rotating is instant and identical from a security standpoint. Then generate a fresh one, paste into .env.local, and redeploy.

Your First Vibe

Why this matters A walkthrough of the entire loop: idea → code → repo → live site. Once you've done it once, every project after this is muscle memory.

Step 09 / AMake a projects folder

mkdir $HOME\dev cd $HOME\dev

Step 09 / BCreate a Next.js app

Next.js is the most-used React framework — Vercel made it, so it deploys there in one click.

npx create-next-app@latest my-first-vibe

Accept all the defaults (TypeScript, Tailwind, App Router — yes to everything).

cd my-first-vibe

code .

Step 09 / DStart the dev server

Inside VS Code's terminal (Ctrl + ` opens it):

npm run dev

Open localhost:3000 ↗. Your app is live on your machine.

Step 09 / EVibe with Claude

Open a second terminal tab (Ctrl + Shift + `) and run:

claude

Try: "Replace the home page with a cosmic-themed landing page for an app called 'StarMail.' Use the existing Tailwind setup." Watch it work. Approve the diff. Refresh the browser.

Step 09 / FAdd polish with shadcn/ui

shadcn/ui is the missing puzzle piece that takes vibe-coded apps from "looks like a default Next.js" to "looks shipped." It's a library of beautifully designed components you copy into your project. Set it up:

npx shadcn@latest init

Accept all defaults. Then add components on demand:

npx shadcn@latest add button card input dialog

Now ask Claude: "Rebuild the home page using shadcn Button and Card components — a hero with a headline, subheadline, and a primary CTA." Browse the full catalog at ui.shadcn.com ↗.

Step 09 / GDebug with DevTools

Press F12 in your browser to open DevTools. Three tabs you'll live in:

  • Console — JavaScript errors land here in red. If your page is blank, check this first.
  • Network — every request your app makes. Click any row to see the data going in and out.
  • Elements — inspect the live HTML and CSS, edit on the fly to test changes.

Workflow when something breaks: copy the red Console error → paste into Claude → "why is this happening, and how do I fix it?" 80% of bugs solved this way.

Hit enter through the prompts. Sixty seconds later, you have a public URL. Send it to a friend.

Chapter 10 · Bonus / When you're ready

Stripe & Resend

Why this matters For your ticketing-app idea, this is the spine. Stripe turns a customer's $35 into a payment in your account. Resend sends them their ticket. Together they handle "money in, ticket out." Skip this chapter for your first throwaway project — come back when you're ready to charge for something real.

Step 10 / ASign up for Stripe

Stripe sign-up ↗

You start in Test Mode by default — identical to live mode except no card actually charges and no money moves. Stay there until launch.

Step 10 / BInstall the Stripe CLI

winget install --id Stripe.StripeCLI
stripe login

Step 10 / CStripe API keys

Dashboard → Developers → API keys. You'll see two:

  • pk_test_… — publishable. Safe in browser code. Use as NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY.
  • sk_test_… — secret. Server-only. Use as STRIPE_SECRET_KEY in .env.local.

Step 10 / DInstall Stripe in your project

npm install stripe @stripe/stripe-js

Step 10 / EWebhooks (the part everyone forgets)

When a payment finishes, Stripe doesn't tell your customer's browser — that could be tampered with. It sends a webhook directly to your server. That's the moment your code mints the ticket and emails it.

To test webhooks locally, run this in a separate terminal tab while your dev server is running:

stripe listen --forward-to localhost:3000/api/webhook

It prints a whsec_… value — that's your STRIPE_WEBHOOK_SECRET for local dev.

Step 10 / FTest cards

In test mode, real cards don't work — and these don't actually charge:

  • 4242 4242 4242 4242 — succeeds
  • 4000 0000 0000 9995 — insufficient funds
  • 4000 0000 0000 0002 — generic decline
  • Any future expiry date, any 3-digit CVC, any ZIP

Step 10 / GSign up for Resend

Resend is a developer-friendly email API — fast, cheap, with a beautiful dashboard. Free tier: 3,000 emails/month, 100/day. More than enough to start a ticketing app.

resend.com/signup ↗

Step 10 / HResend API key & install

Dashboard → API Keys → Create API Key. Copy the re_… value into .env.local as RESEND_API_KEY. Then in the project:

npm install resend

Optional but recommended — write your emails as React components:

npm install @react-email/components

Step 10 / IVerify a sending domain

For real production sending, Resend Dashboard → Domains → Add Domain. They give you DNS records to add wherever you bought the domain (or via Vercel's domains panel). Until verified, use Resend's onboarding@resend.dev sender — it works for testing but only sends to your own verified email.

Step 10 / JThe ticketing flow, stitched

For Griffin's ticketing app, here's the full loop:

  • Customer clicks "Buy Ticket" → redirected to Stripe Checkout (Stripe-hosted page, easiest path)
  • Payment succeeds → Stripe sends a checkout.session.completed webhook to /api/webhook
  • Your code: creates a row in Supabase (event ID, buyer email, ticket ID, QR token), generates a QR code, sends an email via Resend with the QR linked or attached
  • Returns 200 to Stripe so it doesn't retry
  • At the door: scanner hits a /api/verify-ticket endpoint with the QR token, your code marks it used, denies duplicates

Once you've got keys in .env.local, ask Claude to scaffold this whole flow. It's ~200 lines of code you don't have to write.

Step 10 / KGoing live

  • Stripe — toggle to Live Mode (top right of dashboard) → grab pk_live_… and sk_live_… → set as production-only env vars on Vercel. Don't put live keys in .env.local; keep your laptop in test mode forever.
  • Resend — verify your domain, switch the sender from onboarding@resend.dev to tickets@yourdomain.com.
  • Stripe Tax — toggle on in dashboard for automatic sales tax / VAT per buyer location.
  • Stripe Connect — when you onboard third-party event organizers and split each ticket sale, this handles payouts and 1099s. Build it on day 100, not day one.
The big pictureMoney flows in through Stripe, ticket flows out through Resend, record-of-truth lives in Supabase. That's a real product — same architecture as half the SaaS you've ever paid for.

Chapter 11 · Bonus / Level-up your Claude

Power-Ups

Why this matters Two ways to make Claude Code dramatically more capable. Superpowers is a curated set of skills that teach Claude better workflows — planning, debugging, parallel agents, test-driven development. MCP servers connect Claude to outside services (your database, GitHub, Stripe, real-time docs) so it can do work directly instead of asking you to copy-paste.

Step 11 / AInstall Superpowers

Superpowers is a plugin marketplace by Jesse Vincent — it bundles skills like brainstorming, executing-plans, systematic-debugging, test-driven-development, and dispatching-parallel-agents. Run these inside Claude Code (i.e. type claude in your terminal first, then paste each):

/plugin marketplace add obra/superpowers-marketplace
/plugin install superpowers@superpowers-marketplace

Restart Claude Code (Ctrl + C twice, then run claude again). Type / in any session to see the new superpowers:* skills available.

Docs: Plugin discovery ↗ · Marketplace repo ↗

Step 11 / BWhat MCP is, in two sentences

MCP (Model Context Protocol) is a standard for plugging Claude into outside tools. Each "MCP server" is a little adapter — for GitHub, for your Supabase database, for Stripe, etc. — that lets Claude read and write to that service directly. Add servers once with claude mcp add at your PowerShell prompt; from then on, every Claude session can use them.

The --scope user flag below makes each server global (available in every project). Drop that flag for project-only.

Step 11 / CContext7 — live library docs

Pulls up-to-the-minute documentation for any library (React 19, Next.js 15, Supabase, etc.) so Claude isn't relying on its training cutoff. The single highest-leverage MCP for vibe coders.

claude mcp add --scope user --transport stdio context7 -- npx -y @upstash/context7-mcp

Free API key (optional, raises rate limits): context7.com/dashboard ↗

Step 11 / DGitHub — issues, PRs, repos

Lets Claude read and create GitHub issues, pull requests, and repositories directly. Generate a fine-grained PAT first at github.com/settings/personal-access-tokens ↗ with repo read + write scope, then paste it where the placeholder is:

claude mcp add --scope user --transport http github https://api.githubcopilot.com/mcp/ --header "Authorization: Bearer YOUR_GITHUB_PAT"

Step 11 / ESupabase — query your database

Requires Chapter 7 setup. Lets Claude read your tables, propose migrations, and answer "what's in this database" questions without you opening the dashboard.

claude mcp add --scope user --transport http supabase https://mcp.supabase.com/mcp

After adding, run /mcp inside Claude Code → select supabase → Authenticate. A browser tab opens for OAuth.

Docs: Supabase MCP guide ↗

Step 11 / FStripe — products, prices, payments

Requires Chapter 10 setup. Lets Claude create products, generate price IDs, look up customers, inspect payments, and build out your ticketing-app schema directly. Paste your test-mode sk_test_… key where the placeholder is:

claude mcp add --scope user --transport stdio stripe -- npx -y @stripe/mcp --tools=all --api-key=YOUR_STRIPE_SECRET_KEY

Docs: Stripe MCP docs ↗

Step 11 / GPlaywright — browser automation

Lets Claude open a real browser, click around your app, take screenshots, fill forms, and verify behavior. Killer for "did the change actually fix the bug" loops without you babysitting.

claude mcp add --scope user --transport stdio playwright -- npx -y @playwright/mcp

No API key needed — runs locally.

Step 11 / HVerify the fleet

Inside Claude Code, type:

/mcp

You should see your installed servers with green checkmarks. Red? Re-paste the command and check the API key for typos.

How Claude actually uses theseYou don't call MCP tools by name. Just describe what you want — "create a Stripe product called 'GA Ticket' for $35," "show me all rows in the events table," "open the deployed site and screenshot the home page" — and Claude picks the right MCP and uses it. Watch the tool calls scroll past; intervene if it's about to do something destructive.
